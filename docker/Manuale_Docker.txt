🐳 1. Comandi Base Docker
✅ docker --version
Cosa fa: mostra la versione installata di Docker.

Quando usarlo: per verificare che Docker sia correttamente installato e funzionante.

✅ docker pull <image>
Cosa fa: scarica un’immagine dal Docker Hub (o da un altro registry).

Esempio: docker pull nginx

Quando usarlo: prima di eseguire un container basato su un'immagine che non hai ancora.

✅ docker images
Cosa fa: elenca tutte le immagini disponibili localmente.

Quando usarlo: per controllare quali immagini hai già scaricato.

✅ docker ps / docker ps -a
docker ps: mostra solo i container attivi.

docker ps -a: mostra tutti i container, inclusi quelli terminati.

Quando usarlo: per avere il controllo sui container attivi/inattivi.

✅ docker run
Cosa fa: avvia un nuovo container.

Opzioni comuni:

-d: detached (in background)

-p 8080:80: mappa la porta 80 del container alla 8080 del tuo host

--name: assegna un nome al container

Esempio: docker run -d -p 8080:80 --name <name> nginx

✅ Volumi:
-v volume-name:/path/container: volume gestito da Docker

-v /host/path:/path/container: mount di una cartella dal tuo sistema

Quando usarlo: per persistenza dei dati o per modificare file senza rebuild.

✅ docker exec -it <container> bash
Cosa fa: accede all’interno di un container in modalità interattiva.

Quando usarlo: per eseguire comandi o fare debug dall’interno del container.

✅ docker stop <container> / docker start <container> / docker rm <container>
Cosa fanno:

stop: ferma un container in esecuzione

start: riavvia un container fermato

rm: elimina un container

Quando usarli: per la gestione manuale dei container in fase di sviluppo/test.

✅ docker rmi <image>
Cosa fa: elimina un’immagine Docker.

Quando usarlo: per pulire lo spazio su disco o rimuovere immagini non più usate.

🧱 2. Dockerfile
Un Dockerfile è un file di testo che definisce come costruire un’immagine Docker personalizzata.

Esempio base:
Dockerfile

FROM ubuntu
RUN apt update && apt install -y git curl
WORKDIR /workspace
CMD ["/bin/bash"]

Spiegazione:
FROM: base image

RUN: comandi da eseguire durante il build

WORKDIR: cartella di lavoro dentro al container

CMD: comando di default che parte all’avvio del container

✅ docker build -t nome-immagine .
Cosa fa: crea un’immagine a partire da un Dockerfile nella cartella corrente.

Quando usarlo: quando vuoi personalizzare il comportamento di un container o aggiungere tool (come Git, Vim, Curl, ecc.).

📁 3. Volumi e Bind Mount
Volumi Docker:
docker volume ls
per vedere la lista dei volumi

docker run -v volume-html:/usr/share/nginx/html nginx
Gestiti da Docker, salvati in /var/lib/docker/volumes

Persistono anche dopo l’eliminazione del container

Bind Mount:

docker run -v /home/user/html:/usr/share/nginx/html nginx
Collega una cartella locale alla directory del container.

Ideale per lo sviluppo live (modifichi i file localmente e li vedi nel container subito).

Network:
docker network ls
per vedere la list delle reti
